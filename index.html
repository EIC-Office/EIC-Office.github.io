Linus命令

简说Linux

Linux 系统是一个多用户多任务的分时操作系统，要使用Linux系统，就需要登录系统，且必须拥有用户名和密码，用户名和密码通常是系统管理员通过你的申请下发给你，用户的账号一方面可以帮助系统管理员对使用系统的用户进行跟踪，并控制他们对系统资源的访问；另一方面也可以帮助用户组织文件，并为用户提供安全性保护。

初识Linux

多用户的理解:

  • 用户类别:

    • root用户（超级用户）：系统唯一，是真实的。该用户既可以登录系统，可以操作系统任何文件和命令，拥有最高权限.
    • 虚拟用户：这类用户也被称为伪用户或假用户，比如bin、daemon、adm、ftp、mail等；这类用户是系统自身拥有的，而非后来添加的，当然，我们也可以添加虚拟用户.

    • 普通真实用户：这类用户只能登录系统，但只能操作其根目录的内容，权限受到限制，这类用户都算系统管理员自行添加的，比如可能另一位同事需要使用你的电脑，而电脑中有部分文件不便分享，你就可以为他新建一个用户，从而可以限制他的操作权限.

      多用户不仅仅识同一台电脑上的概念，也可以通过远程登陆来进行，比如对服务器的远程控制，不同用户的权限是不同的.

  • 系统通过分组对权限进行管理:(当然我们自己也可以新建分组)

    • Owner:通常为文件创建者，也可以通过chown命令修改.
    • Group:文件与用户公用一些分组，比如文件File 1的Owner为mcyin属于adm组，用户syslog也属于adm组，此时用户syslog对File 1的权限就为Group权限，可以通过chgrp命令修改分组.
    • Other:用户与文件不属于同一分组，此时该用户对文件的权限就为Other权限.

命令总结

1.ls

  • ls:列出当前目录下的文件名称
  • -a:列出所有文件(包括所有隐藏文件)文件名称以及 ’.’ ’..’
  • -A:列出所有文件(包括所有隐藏文件)文件名称
  • -b:把文件名中不可输出的字符用’\’加字符列出
  • -l:目录下文件详细信息
  • -c:根据ctime时间排序显示
  • -t:根据文件修改时间显示
  • -i:输出文件的 i 节点的索引信息
  • -n:格式类似 -l 将其中的Owner与Group用UID与GID代替
  • -o:格式类似 -l显示文件的除组信息外的详细信息
  • -s:输出文件所占内存大小，以及文件名称
  • -u:以文件上次被访问的时间排序
  • -Q:把输出的文件名用双引号括起来
  • -R:列出所有文件名称(包括子目录下的文件名)
  • -S:以文件大小排序
  • -X:以文件扩展名排序
  • -1:以列输出文件名
  • -l:文件详细信息:
    • 第一行:total:当前目录下所有文件所占空间总和(为每个文件所占空间的累加，由于Linux中文件储存以块(4Kb)为单位，所以未满4Kb的所占空间是4Kb，如实际大小为1Kb的文件所占空间为4Kb)
      第一列:文件属性共十个字节(r:可读 w:可写 x:可执行)
      • 第一个字节(特殊表示格): -表示普通文件，d(dirtectory)表示目录，l(link)表示连接文件，b(block)表示块设备文件，c(char)表示字符设备文件
      • 第一组(2-4位):表示Owner权限(使用者权限,根据UID管理)
      • 第二组(5-7位):表示Group权限(同组用户权限,根据GID管理)
      • 第三组(8-10位):表示Other权限(其他用户权限)
    • 第二列:文件个数，普通文件为1，文件夹目录为该文件夹下文件个数
    • 第三列:该文件或目录(文件夹)的拥有者(Owner)
    • 第四列:所属的组(Group)，每一个使用者都可以拥有一个以上的组，不过大部分都是属于同一个(创建者所属的组)，当需要给予使用只特殊权限时，才会给文件另一个组
    • 第五列:文件大小，数字根据单位而不同
    • 第六列:创建日期:–月(月) -(日) –(时):–(分)
    • 第七列:文件名
  • 索引信息(索引节点号码):对于文件除了储存数据，我们还需要找到一个一个地方来储存文件的元信息(如:文件的大小，文件的创建日期等)，这种储存文件元信息的区域叫做inode(索引节点)，每一个索引节点都会有一个号码，操作系统用inode号码来识别不同文件
  • 使用时可能会发现不同的文件名有不同的颜色:
    • 白色：表示普通文件
    • 蓝色：表示目录
    • 绿色：表示可执行文件
    • 红色：表示压缩文件
    • 浅蓝色：链接文件
    • 红色闪烁：表示链接的文件有问题
    • 黄色：表示设备文件
    • 灰色：表示其他文件

2.mv

  • mv file dir:移动文件到指定目录下
  • -b:删除覆盖的目标文件时会先备份
  • -f:强行覆盖同名文件
  • -i:若出现同名文件会先询问用户
  • -u:如果所移动的文件比较新(修改时间比名称相同的文件晚时会覆盖)，反之不会覆盖
  • -t:移动源目录下的文件到目标目录下，也可以用于多个文件向同一目录的移动(属于强制覆盖)
    • 如mv -t Files1 Files2将Files2移动到Files1中
    • 而mv Files1 Files2将Files1移动到Files2中，但mv -t不可用于重命名
    • mv File1 File2 File3 Folder 与 mv -t Folder File1 File2 File3效果相同
  • ../:移动该目录下的所有文件到上一级目录

3.cd

  • cd:进入用户主目录
  • ~ :进入用户主目录；
    • :获取进入此目录之前所在的目录
  • –:进入进去此目录之前所在的目录
  • .. :返回上级目录（若当前目录为“/“，则执行完后还在”/“;”..”为上级目录的意思）；
  • ../.. :返回上两级目录；
  • !$ :把上个命令的参数作为cd参数使用

4.cp

  • cp file dir:复制文件到指定目录下(目录不可以)
  • -a:= -pdr保留权限，复制软链接本身，递归复制
  • -b:删除，覆盖的文件会先备份
  • -d:若来源文件为链接文件的属性(link file)，则复制连结文件属性而非档案本身
  • -r:用递归复制目录或文件到指定目录下(源文件或目录 目标目录)
  • -s:建立软链接，快捷方式(只能在当前目录下进行)
  • -l:建立硬链接
    • 硬链接:通过索引节点号码来指向文件，每一个硬链接可以有多个路径名
    • 软连接:类似于快捷方式，是一种文本文件，包含另一个文件的位置信息
  • -f:强制覆盖相同文件名的文件或目录
  • -i:覆盖文件之前先询问用户
  • -v:显示执行过程
  • -p:保留源文件或目录的属性(包括所有者，所属组，权限与时间等)
  • -P:保留文件目录结构
  • -u(update):更新文件，当源文件比要覆盖的文件新时才会覆盖，否则不覆盖

5.scp

  • scp file Ip:dir:将本地文件复制到相应Ip的目录下
  • -1:强制使用协议1
  • -2:强制使用协议2
  • -4:强制使用IPV4地址
  • -6:强制使用IPV6地址
  • -v:显示执行过程
  • -r:递归复制整个目录
  • -a:将文件目录所有属性保留

使用方法详解:
命令格式

  • scp local_file remote_username@remote_ip:remote_folde
  • scp local_file remote_username@remote_ip:remote_file
  • scp local_file remote_ip:remote_folder
  • scp local_file remote_ip:remote_file
    • 第1,2个指定了用户名，命令执行后需要再输入密码，第1个仅指定了远程的目录，文件名字不变，第2个指定了文件名
    • 第3,4个没有指定用户名，命令执行后需要输入用户名和密码，第3个仅指定了远程的目录，文件名字不变，第4个指定了文件名
  • 从本地复制到远程:
    • scp /home/daisy/full.tar.gz root@172.19.2.75:/home/root
      从远程复制到本地:
    • scp root@/172.19.2.75:/home/root/full.tar.gz /home/daisy/full.tar.gz

12.cat

  • cat 文件名:显示文件内容(文本格式)
  • cat >文件名:创建文件
  • cat File1 File2 >File3:将文件合并为另一文件
  • cat File1 File2 >>File3将文件追加到另一个已存在的文件中
  • cat >>文件名<<EOF:追加内容
  • -n:输出文件内容，并带上每一行的编号(以换行为标准)
  • -b:与-n类似，对空白行不编号(但依然会输出)
  • -s:如果有连续两行以上的空白行，就代换为一行空白行
  • -v:显示执行过程
  • -e:每行结尾加上’$’，(显示不打印文字)加M-oM-;M-?
  • -E:每行结尾加上’$’

13.tac

  • tac 文件名:反向打印(上下颠倒)
  • -i:无视中断信号

14.more

  • more 文件名:查看文件内容
  • +n:从第n行开始显示

  • -c:清屏，从顶部开始显示
  • -p:换行时清屏，与-c效果一样
  • -d:显示完毕后提醒退出操作
  • -s:把连续多个的空行显示为一行

  • +/String:搜寻String字符串位置，从其前两行开始查看

15.less

  • less 文件名:查看文件内容(不会加载整个文件，可以向前或向后移动)
  • -b 数字:设置缓冲区大小
  • -e:文件结束后自动离开
  • -f:强迫打开特殊文件
  • -m:显示百分比(类似more)
  • -N:显示每行行号
  • -o 文件名:将less输出内容在指定文件中储存
  • -s:连续空行合并为一行
  • -x 数字:将缩进显示为规定数量的空格
  • / 字符串：向下搜索“字符串”的功能
  • ? 字符串：向上搜索“字符串”的功能
  • 相关操作
    • n:重复前一个搜索（与 / 或 ? 有关）
    • N:反向重复前一个搜索（与 / 或 ? 有关）
    • b:向后翻一页
    • d:向后翻半页
    • h:显示帮助界面
    • Q:退出less 命令
    • u:向前滚动半页
    • y:向前滚动一行
    • 空格键:滚动一行
    • 回车键:滚动一页
    • [pagedown]:向下翻动一页
    • [pageup]:向上翻动一页

22.find

时间查找
  • -mtime -n/+n:按文件更改时间来查找文件，-n指n天以内，+n指n天以前(不包括第n天)
  • -atime -n/+n:按文件访问时间来查找文件，规则同上
  • -ctime -n/+n:按文件创建时间来查找文件，规则同上
  • -amin -n/+n:查找在指定时间曾被存取过的文件或目录，单位以分钟计算
  • -newer 文件名:以该文件时间为标准，查找比其新的文件并列出
分组及用户查找
  • -uid n:n为数字，这个数字是用户的账号 ID，亦即 UID
  • -gid n:n为数字，这个数字是组名的 ID，亦即 GID
  • -user 账号名称:寻找以该用户为Owner的文件
  • -group 组名:寻找该组下的文件
  • -nouser:寻找档案的拥有者不存在于/etc/passwd 的文件
  • -nogroup:寻找档案的拥有群组不存在于/etc/passwd 的文件
文件参数
  • -name 文件名:按文件名查找，可使用通配符
  • -size [+-]SIZE:搜寻比 SIZE 大(+)或小(-)的档案
    SIZE的单位
    c: 代表 byte
    k: 代表 kbyte
  • -type TYPE:搜寻档案的类型为 TYPE 的文件
    • TYPE的类型主要有：
      • 一般正规档案(f)
      • 装置档案(b, c)
      • 目录(d)
      • 连结档(l)
      • socket(s)
      • FIFO(p)
  • -perm mode:搜寻档案权限等于mode的档案，这个mode为类似chmod的属性值
  • -perm -mode:搜寻档案权限大于mode的档案
  • -perm +mode:搜寻档案权限包含任一 mode 的权限的档案
  • 属性值计算方法:
    • r=4，w=2，x=1
    • 若要rwx属性则4+2+1=7
    • 若要rw-属性则4+2=6
    • 若要r-x属性则4+1=5
其他参数
  • ~:在home中查找
  • .:输出目录
额外的动作
  • -exec 命令:在执行完搜索后继续进行下一个动作，命令形式:命令{} \;(‘{}’代替前面找到的内容)

64.free

  • free:查看内存使用情况
    • Mem:物理内存统计
    • Swap:交换分区使用情况
    • total:表示物理内存总量
    • used:表示总计分配给缓存(包含buffers 与cache)使用的数量，但其中可能部分缓存并未实际使用
    • free:未被分配的内存
    • shared:共享内存，一般系统不会用到
    • buffers:系统分配但未被使用的buffers数量
    • cached:系统分配但未被使用的cache数量
    • available:可使用的内存

  • Cache： 高速缓存，是位于CPU与主内存间的一种容量较小但速度很高的存储器。由于CPU的速度远高于主内存，CPU直接从内存中存取数据要等待一定时间周 期，Cache中保存着CPU刚用过或循环使用的一部分数据，当CPU再次使用该部分数据时可从Cache中直接调用,这样就减少了CPU的等待时间,提 高了系统的效率。Cache又分为一级Cache(L1 Cache)和二级Cache(L2 Cache)，常见的容量有256KB或512KB L2 Cache。

  • Buffer：缓冲区，一个用于存储速度不同步的设备或优先级不同的设备之间传输数据的区域。通过缓冲区，可以使进程之间的相互等待变少，从而使从速度慢的设备读入数据时，速度快的设备的操作进程不发生间断
